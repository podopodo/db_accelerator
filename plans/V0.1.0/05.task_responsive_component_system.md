## Goal

Build reusable components that remain clear under dense data, failure states, and small screens.

## Work

- [ ] Build shell, navigation, global health strip, command bar, context rail, and responsive drawers.
- [ ] Build buttons, fields, filters, tabs, disclosures, dialogs, action sheets, toasts, banners, and progress states.
- [ ] Build status lozenges, metric rows, sparklines, charts, legends, tables, pagination, virtualization, and empty states.
- [ ] Build SQL, identifier, duration, rate, byte, timestamp, error, and copy controls.
- [ ] Build Connection Pressure Map primitives with accessible textual summary.
- [ ] Implement loading, stale, partial, degraded, denied, error, and high-volume variants.
- [ ] Implement keyboard, focus, reduced-motion, high-contrast, and 200% zoom behavior.
- [ ] Build component gallery with deterministic real-domain fixtures.

## Tests

- [ ] Component interaction tests cover mouse, touch, and keyboard.
- [ ] Screenshot tests cover required viewports, themes, density, and states.
- [ ] Long text, large numbers, missing values, and high row count do not break layout.
- [ ] Live updates do not steal focus or move targeted row unexpectedly.

## Done when

- [ ] Components use product tokens, not library default theme.
- [ ] No repeated card pattern replaces proper table or hierarchy.
- [ ] Component gallery passes accessibility and performance budgets.
- [ ] Evidence recorded and status set to `DONE`.
