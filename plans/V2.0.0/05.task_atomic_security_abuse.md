## Goal

Prevent operation IDs and journal features from becoming a privilege, storage, or denial-of-service weakness.

## Work

- [ ] Threat-model ID guessing, namespace probing, hash collision assumptions, result leakage, and replay abuse.
- [ ] Enforce namespace authorization and constant-shape not-found behavior where needed.
- [ ] Limit outstanding work, request bytes, result bytes, journal growth, lookup rate, and conflict rate.
- [ ] Protect journal migration and cleanup permissions.
- [ ] Redact or encrypt sensitive result summaries.
- [ ] Add alerts for ID conflict storms and storage exhaustion.
- [ ] Commission or perform focused API and SQL security testing.

## Tests

- [ ] Cross-tenant enumeration and result-access attempts fail.
- [ ] Oversized and high-cardinality attacks stay bounded.
- [ ] Journal-full condition fails new identified operations without affecting committed data.

## Done when

- [ ] No unresolved critical or high security finding.
- [ ] Abuse limits are documented and visible.
- [ ] Evidence recorded and status set to `DONE`.
