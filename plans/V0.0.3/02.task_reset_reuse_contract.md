## Goal

Prove an upstream connection is clean before another logical client receives it.

## Work

## Evidence

Commit/PR: next session-correctness checkpoint

Commands: The reset path conditionally rolls back an active transaction, restores the probed database/autocommit/charset/collation/time-zone/SQL-mode/isolation/read-write baseline, checks warnings after every reset stage, verifies the final state, and destroys uncertain connections. It retries once on a clean socket. A live MariaDB test contaminates the only physical socket with autocommit, charset, collation, time-zone, SQL-mode, and an uncommitted insert, then proves the next logical client sees the baseline and no row. An injected rollback failure proves two sockets are destroyed. The normal-error, warning-discard, and transaction-rollback continuation lanes also pass. Prepared and temporary state remain refused, so no such handle can reach a reusable socket before their dedicated implementation.
