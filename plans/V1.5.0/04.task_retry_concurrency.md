## Goal

Make duplicate, concurrent, deadlocked, and ambiguous attempts produce one committed outcome.

## Work

- [ ] On existing same ID and hash, return stored committed outcome.
- [ ] On same ID and different hash, reject permanently.
- [ ] Let unique-key locking serialize concurrent first attempts.
- [ ] After winner rollback, allow waiter to execute.
- [ ] After winner commit, make waiter replay winner result.
- [ ] Retry only errors proven to have rolled back the entire transaction.
- [ ] Resolve commit ambiguity from journal before retry.
- [ ] Bound retry count, backoff, lock wait, and total deadline.

## Tests

- [ ] Hundreds of concurrent same-ID requests create one committed effect.
- [ ] Different payload never executes under used ID.
- [ ] Deadlock victim retry preserves one effect.
- [ ] Accelerator restart after commit supports result lookup.

## Done when

- [ ] Duplicate and retry tests pass under race and chaos.
- [ ] Retry decision has stable reason code and audit event.
- [ ] Evidence recorded and status set to `DONE`.
