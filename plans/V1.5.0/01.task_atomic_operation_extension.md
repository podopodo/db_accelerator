## Goal

Freeze how callers submit an identified operation without weakening normal MySQL behavior.

## Work

- [ ] Define versioned atomic batch API carrying namespace, operation ID, isolation, ordered parameterized DML, and requested result summary.
- [ ] Define strict ID syntax, size, uniqueness scope, and caller responsibility.
- [ ] Define canonical payload encoding and server-calculated hash.
- [ ] Define maximum statements, parameters, bytes, runtime, lock wait, and response size.
- [ ] Define accepted DML and forbidden DDL, result sets, external effects, and nontransactional tables.
- [ ] Define retry, timeout, cancellation, and unknown-outcome responses.
- [ ] Evaluate MySQL single-statement attribute or hint extension separately; do not make it part of certified scope unless drivers prove it.

## Tests

- [ ] Order, value, type, namespace, or statement change changes hash.
- [ ] Invalid or oversized request fails before database transaction.

## Done when
