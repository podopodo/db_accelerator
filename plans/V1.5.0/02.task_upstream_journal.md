# Task 02: upstream operation journal

Status: `TODO`

Assignee: `unassigned`

Depends on: `01.task_atomic_operation_extension.md`, `V0.2.0/01.task_schema_catalog.md`

## Goal

Store operation identity and result in the same transactional authority as user data.

## Work

- [ ] Design versioned InnoDB journal schema inside configured upstream database or dedicated schema on same server.
- [ ] Use unique key over caller namespace and operation ID.
- [ ] Store payload hash, contract version, result summary, timestamps, and retention boundary.
- [ ] Keep journal row and user DML in one transaction.
- [ ] Add least-privilege setup and migration commands.
- [ ] Add catalog validation for journal engine, schema, collation, and permissions.
- [ ] Encrypt or minimize sensitive stored result fields by policy.
- [ ] Define retention window and warn that guarantee ends after journal deletion.

## Tests

- [ ] Journal transaction rolls back with user DML.
- [ ] Journal and user DML become visible together after commit.
- [ ] Duplicate namespace and ID blocks or conflicts correctly.
- [ ] Non-InnoDB journal setup is refused.

## Done when

- [ ] Schema migration is restart-safe and downgrade behavior documented.
- [ ] No local file is required to decide committed outcome.
- [ ] Evidence recorded and status set to `DONE`.
