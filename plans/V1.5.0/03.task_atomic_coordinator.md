## Goal

Execute one validated identified operation as one upstream transaction.

## Work

- [ ] Validate catalog epoch, transactional engines, statement class, limits, and trigger/function policy before begin.
- [ ] Acquire one upstream connection and begin requested supported isolation.
- [ ] Insert journal identity and request hash inside transaction.
- [ ] Execute ordered parameterized DML without reordering or splitting.
- [ ] Capture bounded affected rows, insert IDs, and application result payload.
- [ ] Write committed result into journal transaction.
- [ ] Commit once, advance cache generations, then release success.
- [ ] Roll back on definite precommit failure.
- [ ] Resolve uncertain commit through journal lookup and same-ID unique claim before re-execution; journal absence alone is not proof while an old server session could still commit.

## Tests

- [ ] All DML and journal row commit or all roll back.
- [ ] Kill or disconnect at every coordinator state.
- [ ] Cache cannot return pre-operation data after success.
- [ ] Unsupported object is rejected before first user DML.

## Done when

- [ ] Coordinator state machine maps every transition to durable authority and client response.
- [ ] No local-only success state exists.
- [ ] Evidence recorded and status set to `DONE`.
