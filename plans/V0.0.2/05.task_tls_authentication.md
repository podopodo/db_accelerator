## Evidence

Commit/PR:

Security review: Pooled clients now use a separate accelerator username/password secret instead of the upstream credential. The gateway converts the configured password to the double-SHA-1 verifier mandated by `mysql_native_password`, stores only that verifier in the long-lived protocol service, and compares it in constant time; this legacy verifier is not presented as a modern general-purpose password hash. MariaDB 11.7.2 and Oracle MySQL 8.4.10 live integrations prove valid TLS 1.2+ authentication, plaintext downgrade refusal, untrusted-certificate refusal, wrong-password refusal, and rejection of the upstream identity. Certificate files reload on every new TLS handshake; expired/not-yet-valid material fails closed and expiry appears in `/api/v1/status` plus GUI diagnostics. The failed-auth limiter is source-IP keyed, time bounded, and capped at 4,096 entries. `mysql_native_password` remains the only client plugin. The pool's comparable permission key includes client user, upstream endpoint/user, schema, and both transport policies, preventing future multi-pool code from keying by host alone.
