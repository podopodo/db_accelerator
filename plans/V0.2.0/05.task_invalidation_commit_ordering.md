## Goal

Make stale pre-write cache entries unreachable before write success reaches client.

## Work

- [ ] Maintain generation per dependency table.
- [ ] Collect transaction write dependencies without changing statement order.
- [ ] Advance generations only after successful upstream commit.
- [ ] Advance generations before releasing success response.
- [ ] Do not invalidate on rollback.
- [ ] Handle autocommit and explicit transactions.
- [ ] On uncertain dependency, invalidate database-wide or disable cache.

## Tests

- [ ] Reader cannot hit old generation after writer sees success.
- [ ] Rollback keeps valid cache entry.
- [ ] Crash after commit loses RAM cache and restarts cold.
- [ ] Concurrent read/write linearization stress test.

## Done when

- [ ] Cache ordering section of atomicity contract is fully tested.
- [ ] No response path bypasses invalidation barrier.
