Status: `TODO`

Depends on: `02.task_sql_classifier.md`, `03.task_acceleration_policy.md`

## Goal

Serve selected deterministic result sets from RAM without semantic or memory leaks.

## Work

- [ ] Build cache key from normalized SQL, parameters, database, permission scope, relevant session state, schema epoch, and dependency generations.
- [ ] Store canonical column metadata and row values for capability-aware encoding.
- [ ] Enforce global, policy, entry, and result-size byte limits.
- [ ] Add sharding, admission, eviction, TTL, and single-loader stampede control.
- [ ] Stream oversized results direct.
- [ ] Never cache warnings, locks, session effects, transaction reads, or unknown dependencies.

## Tests

- [ ] Key isolation tests for user, database, charset, time zone, and parameters.
- [ ] Memory bound and eviction stress tests.
- [ ] Concurrent miss loads database only within configured collapse rule.

## Done when

- [ ] Cache hit result matches direct database result.
- [ ] Cache can be discarded without correctness or recovery work.
- [ ] Evidence recorded and status set to `DONE`.
